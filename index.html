03
태양광 발전 예측이란 — 실데이터로 보기

실제 발전 데이터가 어떻게 생겼는지, 예측값과 실측값이 어떻게 비교되는지 시각적으로 확인.

① 하루 발전 패턴 — 맑은 날 vs 흐린 날
같은 날 같은 발전소에서 날씨에 따라 발전량이 얼마나 달라지는지. X축: 시간, Y축: 발전량(kW)
② 예측값 vs 실측값 비교
모델이 뱉은 예측선(점선)과 실제 인버터 발전량(실선). 오차가 생기는 구간이 어디인지 확인. 목표 오차율 8% 이내.
③ 일사량(GHI) vs 발전량 상관관계
GHI가 올라가면 발전량도 올라간다. 이 관계가 모델이 학습하는 핵심 패턴.
핵심: 태양광 예측 = "내일 일사량이 얼마일지" 예측 → "그 일사량에서 우리 발전소는 몇 kW 나올지" 계산. 두 단계가 연결된 구조다.

05
데이터 계층 구조

국내 탑 플레이어들이 쓰는 데이터를 정확도 기여도 기준으로 3계층으로 분류.

기본층 — 모든 회사 공통 / 없으면 시작 불가
인버터 실측 발전량
AC/DC 출력, 일누적 발전량. 예측 모델의 정답값(타겟). 이게 없으면 학습 자체 불가.
기상청 수치예보
일사강도·기온·운량·풍속 예보. 예측 시 핵심 입력값. 학습용(실측) + 예측용(예보) 둘 다 필요.
설비 고유 정보
용량·경사각·방위각·위경도. 정적 데이터. 같은 햇빛도 발전소마다 다르게 나오는 이유.
중간층 — 정확도를 올리는 핵심 데이터
현장 센서 데이터
현장 일사량 센서(GHI), 모듈 표면 온도 센서. 기상청 관측값보다 발전소 실제 환경에 정확히 대응.
주변 발전소 비교 데이터
인근 발전소 실측 발전량. 구름 이동 방향 예측과 공간적 상관관계 활용. 단기(1~6h) 정확도 향상.
고급층 — 95%+ 달성하는 차별화 데이터
위성 영상
천리안 2호 실시간 위성 영상. 구름 이동 추적으로 단기 예측 정확도 대폭 향상.
전천카메라 + 패널 이미지
하늘 직접 촬영 + 패널 오염 탐지. 오염 위치 정확도 71.9%, 유형 분류 98% (국내 연구 기준).
해외 NWP + 미세먼지
ECMWF 보조 + 미세먼지 관측·예보. 대기 투과율에 직접 영향.
05-1
발전량 저하 요인 — 예측 정확도를 떨어뜨리는 것들

예측 모델이 아무리 좋아도 이 요인들이 반영되지 않으면 오차가 생긴다. 데이터 수집 단계에서 함께 고려해야 한다.

① 패널 오염
먼지·새 배설물·낙엽 등으로 출력 5~25% 저하. 카메라 이미지 기반 AI로 오염 위치 식별(71.9%)·유형 분류(98%) 가능. Data Augmentation 기법 적용.
비정형 데이터이미지 AI 탐지
② 음영
인근 건물·나무·구조물에 의한 부분 음영. 시간대·계절별로 달라져서 예측이 어렵다. 스트링 전압·전류 데이터로 감지 가능.
string_voltage 이상도메인 룰 탐지
③ 모듈 열화
패널 노후로 인한 점진적 출력 저하. 연 0.5~1% 수준. 장기 데이터에서 추세 분석으로 보정 필요.
장기 트렌드PR 성능비 모니터링
④ 미세먼지
대기 투과율 저하로 GHI 자체가 줄어든다. 에어코리아 PM2.5·PM10 관측·예보 데이터로 보정 가능. 국내 연구과제에서 별도 테이블로 수집.
PM2.5 · PM10에어코리아 API
⑤ 인버터 고장
갑작스러운 출력 0. inverter_status 이상 구간은 이상치로 처리해서 학습 제외. 고장일은 날 전체 제외.
is_valid 플래그학습 제외 처리
⑥ 기상 예보 오차
수치예보 모델 자체의 일사량 예측 오차. 국내 단기예보는 일사량 직접 미제공. 해외 NWP 보조 + 위성 영상으로 보완.
해외 NWP 보조위성 영상 보완
데이터 수집 관점에서: 패널 오염·음영은 이미지 + 스트링 전압 데이터로, 미세먼지는 에어코리아 API로, 기상 예보 오차는 해외 NWP + 위성으로 각각 대응한다. 이 요인들을 데이터로 확보하는 것 자체가 예측 정확도 향상의 핵심이다.
06
원천데이터 — 필요한 자료 상세 정의

현재 조사 중인 단계. 가장 기본이 되는 3가지 원천데이터.

▶ 현재 조사 중 — 원천데이터 종류·항목·우선순위
원천데이터는 ① 발전소 실측 / ② 기상 데이터 / ③ 설비 고유정보 3가지.
①이 없으면 학습 불가, ②가 없으면 예측 불가, ③이 없으면 발전소별 특성 반영 불가.

① 발전소 실측 데이터 — 예측 모델의 정답값(타겟)

국내 디지털트윈 연구과제 기준: 2년간 155만 9천 건 수집. 발전소당 시간 단위로 적재. 설비이용률 기준 이상 구간만 학습에 사용.

모델 학습 및 예측 — 방법론 정리

수집·가공된 데이터를 AI 모델에 넣어 패턴을 학습시키고 예측값을 뽑아내는 단계. 현재는 데이터 수집 완료 후 진행 예정.

핵심 원칙: 태양광 발전량 예측은 시계열 회귀(regression) 문제. 사람이 라벨을 붙이는 작업이 아니라, 인버터 실측 발전량이 자동으로 정답값(타겟)이 되는 구조. 모델이 좋아도 데이터 품질이 나쁘면 정확도가 안 나온다 — 데이터가 먼저.
머신러닝 기반
XGBoost · LightGBM
트리 기반 앙상블 모델. 빠르고 정확도가 높아 국내 연구과제에서 기본으로 사용. 피처 중요도를 직접 확인할 수 있어서 어떤 변수가 발전량에 영향을 주는지 파악 가능. 데이터가 적을 때도 잘 작동.
적합한 상황
초기 베이스라인 모델 · 피처 중요도 분석 · 데이터 부족 구간
딥러닝 기반
LSTM
시계열 딥러닝. 과거 발전 패턴의 흐름을 기억하는 구조. "1시간 전이 이랬으면 지금은 이럴 것"을 학습.
CNN-LSTM
CNN으로 시계열의 지역 패턴을 추출한 뒤 LSTM으로 흐름을 학습. 일사량 변화 패턴 포착에 효과적.
Multi Input Model
발전소 실측 / 기상 / 설비정보를 각각 별도 입력으로 받는 구조. 데이터 종류가 다양한 우리 프로젝트에 적합.
검증 방법 — Walk-Forward Validation
시계열 데이터는 랜덤 분할 절대 금지. 미래 데이터로 과거를 예측하는 데이터 누수(leakage)가 발생해 실제보다 정확도가 높게 나온다. 반드시 시간 순서를 유지한 walk-forward 방식으로 검증해야 한다.
