수집 기준 근거 — 왜 2년·15분·15개소인가

2년: 기상 요인(계절·일사량·운량 등)에 따른 발전량 변동성 반영 목적. 1개년 데이터는 사계절 패턴이 1회에 불과해 검증 데이터가 부족하고 학습/검증 분리가 어려움. 2개년 확보 시 1년차 학습, 2년차 검증으로 예측 모델의 일반화 성능 검증이 가능하다.

15분: 구름 이동 등으로 인한 10~30분 단위의 단기 출력 변동성 반영 목적. 급격한 변동인 램프 이벤트(Ramp Event)는 1시간 단위 분석 시 정보가 평활화(유실)됨. 스위스 국가 연구(6,434개 PV 분석, 2026)에서도 램프 이벤트 분석 및 전력시장 기준 해상도로 15분 단위를 제시했다.

15개소: 사계절 반복성, 단시간 변동성, 발전소 간 다양성을 동시에 확보하기 위한 최소한의 운영형 데이터 수집 기준이다.

참고문헌: 스위스 6,434개 PV 발전소 데이터 기반 램프 이벤트 연구 (2026) ↗

02
태양광 발전 예측이란 — 패턴으로 이해하기

실제 발전 데이터가 어떤 패턴을 갖는지 시각적으로 확인한다. 아래 차트는 일반적인 태양광 발전 패턴을 표현한 시뮬레이션 예시다.

① 하루 발전 패턴 — 맑은 날 vs 흐린 날
날씨에 따라 발전량 차이가 3배 이상 발생. 구름 유입 여부가 핵심 변수.
※ 시뮬레이션 예시 — 실제 발전소 데이터 수집 후 교체 예정
② 예측값 vs 실측값 — 허용 오차 범위 비교
개정 전(8%, 회색 점선)보다 개정 후(6%, 주황 점선) 허용 범위가 좁아짐. 예측값이 주황 점선 안에 들어와야 최고 정산금(4원/kWh) 수령 가능. ※ KPX 제24-6차 전력시장운영규칙 개정 — 태양광 2025.06.01 적용
※ 시뮬레이션 예시 — 목표 오차율 6% 이하 (KPX 제24-6차 전력시장운영규칙 개정, 태양광 2025.06.01 적용)
③ 일사량(GHI) vs 발전량 상관관계
GHI가 높을수록 발전량 증가. 모델이 학습하는 핵심 패턴. 고온·미세먼지 구간에서 관계가 약해짐.
※ 시뮬레이션 예시 — 실제 데이터 수집 후 상관계수 분석 예정
구조 요약: "내일 일사량(GHI)이 얼마일지" 예측(기상 모델) → "그 일사량에서 우리 발전소는 몇 kW 나올지" 계산(발전 모델). 패널 오염·음영·미세먼지는 두 번째 단계에서 오차를 만드는 주요 요인이다. ⚠ 2025년 6월부터 태양광 예측오차율 기준 강화 — KPX 제24-6차 전력시장운영규칙 개정으로 정산금 수령 기준이 기존 8%에서 6% 이하로 낮아짐. 예측 모델의 정밀도 요건이 직접 높아진 상황이다.
자료: KPX 제24-6차 전력시장운영규칙 개정 공고 ↗

08
모델 학습 및 예측 — 방법론 정리

모델 파트의 목적은 특정 알고리즘을 먼저 정하는 것이 아니라, 데이터 수준·예측 horizon·운영 목적에 맞는 모델 선택 기준을 도출하는 것이다.

핵심 원칙: 태양광 발전량 예측은 시계열 회귀 문제이며 정답값은 인버터 실측 ac_power다. 국내 실험 논문 분석 결과 공통으로 내린 결론 —
어떤 데이터를 어떻게 만들어 넣느냐가 모델 선택보다 중요하다. 모델을 먼저 확정하지 않고 단계별로 검증하는 이유가 여기 있다.
목적
모델 벤치마킹의 목적
데이터 수준 판단
현재 확보 가능한 실측·기상·설비정보로 당장 가능한 모델과, 1~2년 데이터 축적 후 가능한 모델을 구분한다.
Horizon별 모델 선택
15분 단기 변동, 1~6시간 운영, 하루 전 제출은 필요한 입력과 모델 구조가 다르므로 별도 비교가 필요하다.
운영 적용성 검토
정확도뿐 아니라 설명 가능성, 재학습 주기, API 장애 대응, EMS/ESS 연동 가능성까지 함께 본다.
개념
NWP (수치예보) — 기상 입력 데이터의 핵심
수치예보(Numerical Weather Prediction, NWP)란 대기 현상의 역학·물리 원리에 대한 지배방정식을 컴퓨터로 수치 적분하여 미래 대기 상태를 정량적으로 예측하는 기술이다. 태양광 발전량 예측에서 NWP는 GHI·기온·운량·풍속·습도 등을 시간 단위로 예보하는 핵심 입력 소스다.
학습 시 — 과거 실측값 사용

GHI · 기온 · 풍속 · 전운량 · 습도 · 강수
→ 실제 기상과 발전량 관계 학습
예측 시 — NWP 예보값 사용
기상청 KIM · ECMWF · Open-Meteo
48h 앞 GHI·기온·운량 예보

주의: 기상청 단기예보 API는 일사량(GHI)을 직접 제공하지 않는다. Open-Meteo의 shortwave_radiation으로 보완이 필요하다.

08-1
모델 진화 흐름 — 고전에서 현재까지

태양광 발전량 예측 모델은 "무엇을 입력으로 넣느냐"가 계속 넓어지는 방향으로 발전해왔다. 이 흐름을 이해해야 왜 지금 우리가 쓰려는 모델이 적합한지 근거가 생긴다.

① 고전 통계 모델 — ARIMA, AR, MA
발전량 데이터 하나만 입력. "과거 패턴이 반복된다"는 가정으로 예측한다.
한계: 기상 변수를 전혀 반영하지 못함. 구름이 갑자기 끼면 예측값이 실측과 크게 벗어남. (Cheng et al., 2012)
② 기상 변수 발견 — ANN, KNN
온도·습도·운량을 독립변수로 추가. 발전량 예측 정확도가 의미있게 향상됐다.
발견: 모듈 온도가 발전량에 선형적으로 영향. 구름 있을 때 일반 모델은 과대 예측. 습도 증가 시 일사량 감소. (Skoplaki & Palyvos, 2009 외)
③ 트리 기반 ML — Random Forest, XGBoost, LightGBM
비선형 관계 학습 가능. 피처 중요도로 어떤 변수가 발전량에 얼마나 영향을 주는지 확인 가능하다.
국내 21개 발전소 NWP 기상 예보 입력으로 여러 ML 모델 비교 — 고전 통계 모델 대비 정확도 향상 확인. (이용택 외, 대한산업공학회지 2021)
④ 시계열 딥러닝 — LSTM, BiLSTM, GRU
시간 흐름의 패턴을 자동 학습. 과거 발전량 흐름을 기억하면서 예측한다.
주의: LSTM 성능이 낮게 나오는 경우는 모델 문제가 아니라 야간 데이터 제거 시 시퀀스 끊김 문제다. 전처리 방식이 성능을 결정한다. (이용택 외, 2021)
⑤ 피처 확장 + 병렬 모델 — DBiLSTM, CNN+LSTM
기상 데이터 8개를 특성 공학으로 494개까지 확장. DNN+BiLSTM 병렬 결합으로 단일 모델 대비 RMSE 15~37% 향상됐다.
핵심: 어떤 데이터를 어떻게 만들어 넣느냐가 모델 선택보다 중요하다. GHI 상관계수 0.704로 가장 강한 입력 변수 확인. (황주훈·김창복, Journal of KIIT 2024)
⑥ 현재 — 하이브리드 (물리 + 데이터)
pvlib 물리 모델로 클리어스카이 기준선을 잡고, ML/DL로 실제 발전량과의 차이(잔차)를 학습한다. 설명 가능성과 정확도를 동시에 확보하는 방향이다.
글로벌: 하이브리드 모델이 단일 DL 대비 25~30% 정확도 향상. (ScienceDirect 재생에너지 예측 리뷰, 2024)
핵심 인사이트: 모델이 발전할수록 "입력 데이터의 종류"가 늘어났다. 발전량만 → 기상 추가 → 피처 확장 → 다중 입력 통합. 더 좋은 모델을 쓰려면 더 많은 종류의 데이터가 먼저 있어야 한다. 우리가 데이터 수집을 먼저 하는 이유가 여기 있다.

후보 모델 상세 비교 — 국내 논문 근거

국내 실험 논문 분석 결과를 기반으로 우리 프로젝트 적용 근거를 도출했다.

① 단일 최고 모델은 없다
발전소 위치·기상 패턴·데이터 특성에 따라 최적 모델이 달라진다. 국내 21개 발전소 실험에서 부산은 XGBoost, 영월은 Random Forest, 삼척은 SVR이 각각 최고 성능을 보였다. 전국 통합 모델 하나로 모든 발전소를 커버하려 하면 안 된다는 근거다. (이용택 외, 대한산업공학회지 2021)
② 전처리·피처 설계가 모델 성능을 결정한다
LSTM이 XGBoost보다 낮게 나온 건 모델 자체 문제가 아니라 야간 데이터 제거 시 시퀀스가 끊기는 전처리 문제였다. 기상 데이터 8개를 특성 공학으로 494개까지 확장했을 때 RMSE가 0.0907 → 0.0793으로 향상됐다. lag·rolling·sin/cos 인코딩 같은 피처 설계가 모델 선택만큼 중요하다. (황주훈·김창복, Journal of KIIT 2024)
③ 병렬 모델이 단일 모델보다 좋다
DNN의 각 시점 특성 추출 능력과 BiLSTM의 시계열 패턴 학습 능력을 병렬로 결합한 DBiLSTM 모델이 단일 BiLSTM 대비 RMSE 기준 15~37% 향상됐다. 깊은 레이어 없이도 Vanishing Gradient 문제를 피하면서 높은 성능을 낼 수 있다. (황주훈·김창복, Journal of KIIT 2024)

미세먼지 데이터 수집이 필요한 이유
PM10·PM2.5 미세먼지 농도가 높아질수록 대기 투과율이 떨어져 GHI가 감소하고 발전량이 줄어든다. 미세먼지 데이터를 입력에 포함하면 예측 정확도가 향상된다는 것이 국내 연구에서 확인됐다.
일반 발전소
미세먼지 농도 상승 → GHI 감소 → 발전량 감소. 에어코리아 실측·예보 수집으로 보정 가능하다.
울산 임해공단 특성
공업단지 특성상 미세먼지·해풍 영향이 일반 지역보다 크다. 에어코리아 PM2.5·PM10 수집이 필수적이다.
출처: PM10 미세먼지 데이터를 고려한 기계 학습 기반 태양광 발전 효율 예측 (국내 연구 — DOI 확인 중)
울산 발전소 적용 방향: 논문 연구의 경우 변수를 제한해서 모델을 비교하지만, 실서비스로 운영하려면 더 많은 입력이 필요하다. NWP 예보에 더해 발전소 실측 lag, 설비 고유정보, 미세먼지 데이터를 함께 넣는 구성이 필요하다.
기본 입력: NWP 예보 (GHI·기온·운량·풍속) + 발전소 실측 lag + 설비 고유정보 (용량·경사각·방위각)
울산 추가 입력: 울산은 임해 공업단지 특성상 미세먼지·해풍 영향이 일반 지역보다 크다. PM2.5·PM10 실측·예보 데이터를 입력에 포함하는 것이 필요하다.
검증 방식: 이론으로 모델을 미리 확정하지 않고 → 1단계 XGBoost 베이스라인 → 발전소별 XGBoost / RF / LSTM 병렬 비교 → 울산 데이터로 직접 검증해서 선택한다.

학습 데이터셋 구조

원천데이터 → 전처리·가공 → 학습 데이터셋 3단계.

STEP 1. 원천데이터 수집·적재
7개 테이블 분리 적재. 발전소 인근 시군구 기준 기상 매핑.

15분 단위 수집
기상 데이터 (5종 테이블)
ASOS · NWP · 미세먼지 · 예보
1시간 단위 수집
15분 단위 (인버터)1시간 단위 (기상)총 목표 약 190만 건
STEP 2. 전처리 + 피처 가공
15분→1시간 리샘플링 후 기상 데이터 JOIN. 파생 컬럼 생성.
리샘플링 15min→1h결측치 보간이상치 3σlag_1h · lag_24hrolling_mean_6hsin/cos 인코딩발전량 0~1 스케일링
발전소별 vs 통합 모델: 데이터 충분한 발전소는 맞춤형 모델이 더 정확하다. 신규 발전소는 0~1 스케일링 후 통합 모델 적용. VPP 운영 시 통합 모델 필수.
STEP 3. 품질 관리 — 학습 데이터 선별
사람 손이 유일하게 가는 단계. 설비이용률 기준 이하 구간 학습 제외.
성능 평가 지표
설비이용률(%) = 계량발전량 ÷ 설비용량 × 100
→ 정상 범위 구간만 학습 데이터로 사용
예측오차율(%) = |예측발전량 − 실제발전량| ÷ 설비용량 × 100
→ 1차 목표: 6% 이내(정확도 94%+) · 고도화 목표: 4% 이하(정확도 96%+)

  • Open-Meteo 테스트 — 울산 위경도 기준 GHI 예보값 응답 확인
